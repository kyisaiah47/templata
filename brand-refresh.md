# Templata Brand Refresh

## Core Problem
Current site looks like an art project, not a SaaS product. Messaging is vague and confusing. Users don't understand what it does or how to use it.

## New Positioning

**Core tagline:**
```
Wikipedia × Notion for life planning
```

**What it actually is:**
A community-driven library of expert frameworks and guides for major life decisions and transitions. Like Wikipedia, it's curated and collaborative. Like Notion, it's structured and actionable. Not journaling/reflection—actionable playbooks.

**One-sentence pitch:**
"Expert frameworks for life's biggest decisions—career changes, relationships, health crises, major transitions. Get the guide, not a blank page."

**Evolution path:**
- **Phase 1** (Now): Curated library of expert-created guides
- **Phase 2** (Next): User-requested guides + community discussions
- **Phase 3** (Future): Expert partnerships + verified content

## Contextual Article Suggestions (CRITICAL UX IMPROVEMENT)

### The Problem:
Currently, articles are just listed at the template level. But a "Wedding Planning" guide might have 50 prompts and 20 articles. Users don't know which articles are relevant to which prompts.

Example:
- Prompt: "How do I choose a wedding photographer?"
- Relevant article: "10 Questions to Ask Wedding Vendors"
- Problem: That article is buried in a list with 19 others about catering, budgets, venues, etc.

### The Solution:
**Prompt-level article suggestions** — Show 1-3 relevant articles right next to each prompt.

---

### Implementation Approach

**Database schema addition:**

```sql
-- Add a junction table linking prompts to articles
CREATE TABLE prompt_articles (
  id uuid PRIMARY KEY,
  prompt_id uuid REFERENCES template_prompts,
  article_id uuid REFERENCES template_articles,
  relevance_score int DEFAULT 1, -- for sorting
  created_at timestamp DEFAULT now()
);

-- Or simpler: add article_ids array to prompts
ALTER TABLE template_prompts
ADD COLUMN suggested_article_ids uuid[];
```

### How to Populate This Data

**Option 1: Manual curation (slow but accurate)**
- You manually tag which articles go with which prompts
- Time-consuming but highest quality

**Option 2: AI-assisted matching (fast, good enough)**
- Use Claude Code to analyze each prompt
- Match it to relevant articles based on semantic similarity
- Generate `prompt_articles` relationships automatically
- You review and adjust

**Example Claude Code script:**

```typescript
// Pseudo-code for matching prompts to articles
for each template:
  for each prompt in template.prompts:
    // Ask Claude to find most relevant articles
    const matchedArticles = await claude.analyze({
      prompt: `
        Given this question: "${prompt.text}"

        Which of these articles are most relevant (pick top 2-3):
        ${template.articles.map(a => `- ${a.title}: ${a.summary}`).join('\n')}

        Return article IDs in order of relevance.
      `
    })

    // Store relationships
    await db.insert('prompt_articles', {
      prompt_id: prompt.id,
      article_ids: matchedArticles
    })
```

**Option 3: Hybrid approach (best)**
- AI generates initial matches
- You review and refine via simple admin UI
- Flag obviously wrong matches
- Add missing connections

### The Flow

**1. User requests a guide**
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[Submit request]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**2. Request goes to queue**
```sql
CREATE TABLE template_requests (
  id uuid PRIMARY KEY,
  user_id uuid REFERENCES auth.users (optional),
  email text,
  topic text NOT NULL,
  category text,
  status text DEFAULT 'queued', -- queued, processing, ready, published, merged
  votes int DEFAULT 1,
  created_at timestamp DEFAULT now(),
  published_template_id uuid REFERENCES templates
);
```

**3. Nightly generation (your local Claude Code script)**
- Pulls top-voted queued requests from DB
- Generates template using your existing Claude Code workflow
- Validates quality (structure, completeness, accuracy)
- Checks for duplicates (hash matching)
- Inserts into templates table with `source='community-requested'`
- Updates request status to 'ready' or 'published'

**4. Attribution**
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[At top of community-requested templates]

📝 Community Template
Requested by 12 users • Created [date] • Curated by Templata

[👍 Helpful] [💬 Discuss] [✏️ Suggest edit]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**5. User notification**
```
Email:
Subject: Your template request is ready!

Hi there,

The template you requested ("Starting a side business while working full-time")
is now live on Templata.

[View template →]

Thanks for helping us grow the library!
```
